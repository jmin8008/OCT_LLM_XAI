## ⚠️ 0-0. 전제 정정 (2026-06-02, fluid mask 재구축 후)

**기존 §0의 FER=0.028은 "상수 가짜 마스크" 아티팩트였음이 확인됨.**
FER 계산에 쓰이던 마스크가 모든 eye에 대해 동일하게 8×8 격자 하단-중앙
(`fluid_mask[4:8,2:6]=1`, OCT에서는 맥락막/배경 영역)에 고정돼 있었고,
eye별 실제 유체 위치(상단/중단/망막 띠 내부 등 천차만별)를 전혀 반영하지 못했다.

→ Claude-vision으로 **eye별 실제 유체 마스크(6×6)** 를 직접 주석하고, **36 eye 전부의 raw
attention map을 GPU로 재생성**(`gen_rollout_realmask.py`, Z1, query=마지막 토큰)해 재계산:

```
FER (상수 가짜 마스크)  평균 0.018
FER (실제 유체 마스크)  평균 0.210   ← 약 11.4× 상승, 최대 0.42 (115R)   [34 fluid eyes]

Spearman r(FER_real, KG-align Z1) = -0.051  p=0.77   ← 여전히 무상관
  FER|aligned = 0.198 (n=20)   vs   FER|misaligned = 0.212 (n=15)   (차이 없음)
```

**두 가지 결론 (둘 다 중요):**
1. **FER 11× 상승** → RetinaVLM은 **실제로 유체를 본다.** "모델이 유체를 안 본다(text-bias)"는
   아래 §0의 *메커니즘 진단은 틀렸다* — 마스크 위치 아티팩트였다.
2. **그런데 FER↔KG-align은 실제 마스크로도 여전히 무상관** (r≈-0.05). 즉 *눈은 떴지만,
   "얼마나 보는지"가 "임상적으로 맞게 말하는지"와 연결되어 있지 않다.* 시각-언어 **분리(decoupling)**
   현상 자체는 살아남았고 오히려 더 깨끗해졌다.

**재정의된 가설 (더 강해짐):** 문제는 "보게 만들기"가 아니다(이미 봄). instill의 목표는
**decision 시점에 시각 grounding ↔ 텍스트 결정을 *인과적으로 결합(wiring)*** 하는 것이다 —
"보는 것"이 "말하는 것"을 *구동(drive)* 하도록.

- 마스크 데이터셋·제작법·한계: [`fluid_masks/FLUID_MASKS.md`](fluid_masks/FLUID_MASKS.md)
- 36-eye 재계산 결과: `code/xai_e3_tier3_rollout_realmask.json`, 노트북 §1b/§1c
- 시각 검증: `fluid_masks/ALL_overlays_montage.png`

**남은 caveat:** 이 FER은 prompt 마지막 토큰(query=-1) 기준 — *생성 시점*("continue"/"stop"
토큰)이 아니다. `mini_gpt4.attention()`은 forward-only라 생성-토큰 attention은 별도 캡처 필요(Stage 1).

```
현재 RetinaVLM (tier3, Z1) 결과:
  FER (유체 영역 attention 비율) = 0.028 ± 0.006   ← ⚠️ 상수 가짜 마스크 기준 (실제 마스크 ≈0.20)
  KG-alignment (text) = 0.571
```

## 1. 가설 정의

> **"VLM의 내부 Cross-Attention 레이어가 임상 토큰('Continue', 'SRF', 'fluid') 생성 시점에
> KG가 지목한 결정-바이오마커 영역(IRF/SRF)을 수학적으로 강제 참조(Look)하도록
> 시각 정렬(Visual Grounding)을 주입(Instill)하면,
> 최종 텍스트 답변의 KG 정합도(Fidelity)가 필연적으로 동반 상승한다."**

### 가설의 논리 구조

```
[현재]
  이미지 → (text-bias) → "continue" 토큰
         유체 안 봄 ↑           KG 정합: 우연 (0.571)

[Instill 후]
  이미지 → (fluid-aligned attention) → "continue" 토큰
         유체 봄 ↑ (FER↑)             KG 정합: 필연 (0.571 → ?)
```

### 실험 대상 모델: RetinaVLM (tier3)
- 이유: attention 접근 가능 (mini_gpt4.attention(), rollout.py), 현재 FER=0.028으로 개선 여지 가장 큼
- 비교군: 동일 모델 instill 전후 + Qwen3.6+Z2_KG_COT (텍스트-레벨 instill 기준선)

## 2. 올바른 "XAI Instill"의 메커니즘 (AGE-VLM 원문 기반)

```
잘못된 이해 (기각):
  GradCAM heatmap을 이미지 위에 그려서 VLM에 다시 입력 (이미지 수정)
  → Inference-time image manipulation, 모델 내부와 무관

올바른 이해 (채택):
  1. SAM/CNN으로 병변 공간 지식(fluid mask) 추출
  2. VLM의 Cross-Attention 가중치 자체를 병변 영역으로 강제 정렬
  3. 학습 시점에 Attention-guidance loss로 내면화
```

**AGE-VLM 원문:**
> *"interleaved cross-attention layers to instill vision capabilities...
> enforces in VLM the ability 'look' at the correct image regions
> by leveraging spatial knowledge distilled from SAM"*

## 3. 실험 프로토콜 (3단계)

### Stage 1: 토큰-조건부 Attention-KG 상관관계 분석 [E3d] — **지금 실행 가능**

**목표:** "continue" 토큰 생성 순간의 attention map이 KG가 지목한 유체 영역과 얼마나 일치하는가?
이것이 기존 GradCAM(시각 인코더 활성화)과 어떻게 다른가?

**방법:**
1. RetinaVLM `inner.attention()` → `rollout_from_retinavlm()` → 토큰별 attention map 추출
2. "continue" 토큰 위치의 attention vs "stop" 토큰 위치의 attention 비교
3. 각 attention map의 FER 계산
4. 이 FER ↔ 해당 예측의 KG-alignment 상관관계 (Spearman r)

**기대 결과:**
- H0: "continue" 토큰 생성 시 FER과 KG-align 여전히 무상관 → text-bias 확인
- H1: "continue" 토큰 시 FER이 "stop" 토큰 때보다 높음 → 모델이 어느 정도 fluid 반응

**✅ 실행 결과 (2026-06-02, `gen_stage1_token_attn.py`, 35 eye):**
> `mini_gpt4.attention()`이 forward-only라 `answer_preamble`로 결정 단어를 teacher-force해
> 그 토큰의 rollout-attention FER(실제 마스크)을 paired 비교.
>
> | | FER @ "continue" | FER @ "stop" | Δ(cont−stop) | Wilcoxon p |
> |---|---:|---:|---:|---:|
> | mean | **0.2051** | **0.2047** | **+0.0004 (0.31%)** | 3.65e−5 |
>
> - **H1이 *기술적으로만* 성립:** 방향은 일관(모든 eye에서 continue가 미세하게 높음)되어 p<1e-4로
>   유의하지만, 효과크기(0.3%, max 0.0016)는 **무시 수준.** → image attention은 거의 **decision-invariant**.
> - FER@natural-decision ↔ KG-align: r=−0.063, p=0.72 (무상관 재확인).
> - ci_pred **35/35 continue** (stop 0개, CI-AUC≈chance) — 결정 자체가 시각과 무관히 고정.
>
> **결론:** 모델은 유체를 보지만(FER≈0.21), 그 grounding은 임상 결정과 **인과적으로 분리**돼 있고,
> 결정 토큰을 바꿔도 거의 움직이지 않는다. instill의 표적 = 이 **0.3% vestigial 결합을
> 결정-구동 수준으로 증폭**(Stage 2/3). 결과: `code/xai_stage1_token_attn.json`, 노트북 §2b.

**구현:** `5_age_vlm_xai_instill.ipynb` §2/§2b + `code/gen_stage1_token_attn.py`

### Stage 2: Inference-time Attention Steering (학습 없는 근사) — **지금 실행 가능**

**목표:** 학습 없이 추론 시점에 attention 편향을 주입 → KG-align 변화 측정

**방법:**
외부 Task 1 CNN(fluid biomarker 분류기, AUC=0.9636)으로 fluid 영역 feature를 추출,
이를 RetinaVLM의 LLM 레이어 cross-attention에 **bias(additive)** 로 주입:

```python
# Hook: image token positions corresponding to fluid regions get positive bias
def attention_bias_hook(module, input, output):
    attn_weights = output[0]   # [B, H, T_q, T_k]
    # fluid_token_mask: [T_k] — 1.0 for fluid-region image tokens
    attn_weights[:, :, :, fluid_token_mask > 0] += BIAS_STRENGTH
    return (attn_weights,) + output[1:]
```

`fluid_token_mask`는 **Claude-vision 실제 마스크**(Task1 CNN GradCAM 대신)로 6×6→36 image token 매핑.

**✅ 실행 결과 (2026-06-02, `gen_stage2_attn_steer.py`, 35 eye):**
> transformers 5.3.0 `eager_attention_forward`를 monkeypatch해 pre-softmax 점수의
> fluid image-token key 열(pre_len=41+fluid_idx)에 **+bias** 주입(generate 전 디코드 스텝 적용).
> bias 0/3/6 스윕.
>
> | bias | resolved | uncertain | cont | stop | CI-AUC | flip→stop |
> |---:|---:|---:|---:|---:|---:|---|
> | 0 | 35 | 0 | 35 | 0 | — | — |
> | 3 | 14 | **21** | 11 | 3 | **0.312** | 3건 **전부 오답** |
> | 6 | 1 | **34** | 1 | 0 | — | 모델 파괴 |
>
> - **Steering 작동:** FER 0.204→0.955 (4.7×). attention은 완전히 제어 가능.
> - **결정도 움직임(고정 아님):** bias3에서 3 eye가 continue→stop. → LLM 결정은 시각 attention으로
>   *움직일 수 있다*.
> - **그러나 전부 열화:** flip 3건(45L/273L/345R) 모두 오답(y_continue=1), CI-AUC 0.312(<chance),
>   60% uncertain(문장 붕괴), bias6은 34/35 uncertain. 209R(무유체)=불변(음성대조 OK).
>
> **결론:** inference-time steering으로는 decoupling을 **고칠 수 없다.** "보게 강제"하면 틀린 결정·비문만
> 유발 — 시각 grounding이 올바른 임상 논리를 담고 있지 않기 때문(Stage 1과 일치). → instill은
> **training-time(Stage 3)** 이어야 하며, fluid-grounded attention을 *일관된 추론으로 통합*하도록
> 학습(LM coherence loss + attention guidance 동시)해야 함이 실증됨.
> 산출: `predictions_tier3_Z1_instill_b{0,3,6}.json`, `xai_stage2_fer_steer.json`, 노트북 §3b.

**구현:** `5_age_vlm_xai_instill.ipynb` §3/§3b + `code/gen_stage2_attn_steer.py`

### Stage 3: Training-time AGE-VLM Full Method — **향후 과제 (학습 필요)**

**목표:** AGE-VLM 원문의 방법론 완전 이식

**방법:**
1. **교사 모델 (Teacher):** Task 1 CNN (보유, 검증됨) → fluid region 공간 마스크 생성
   - 기존 `experiments_product/` KAD Knowledge Encoder와 결합 가능
   - SAM을 직접 사용하는 것의 의료 도메인 대안
2. **Attention-guidance loss:** VLM Cross-Attention 가중치와 fluid 마스크 간 KL-divergence
   ```
   L_total = L_LM + λ · L_attn_guide
   L_attn_guide = KL(attn_cross || fluid_mask_normalized)
   ```
3. **LoRA 미세조정:** Cross-Attention 레이어만 학습 (나머지 frozen)
   - Forgetting을 최소화하면서 fluid grounding만 주입
   - APTOS-2021 학습셋 221 eyes 활용

**기대 결과:**
- FER: 0.028 → (0.1~0.3 목표)
- KG-align: 0.571 → (0.7~0.85 목표)
- AMD staging 성능은 유지 (catastrophic forgetting 방지)

**구현:** 별도 학습 노트북 (VRAM H100 80GB 필요)
