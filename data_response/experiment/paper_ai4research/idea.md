# Idea Summary: Closing the Loop — End-to-End Autonomous ML Research with Open-Source Science Agent, AutoResearch, and Automated Manuscript Generation

## Core Contribution

We present a fully autonomous end-to-end research pipeline that couples three components: (1) an **Open-Source LLM Science Agent Harness** built on OpenClaude (a multi-provider Claude Code derivative), (2) an **AutoResearch** loop that iterates hypothesis → implementation → execution → analysis without human intervention, and (3) **Paper Orchestra**, a multi-agent LaTeX manuscript generation system that converts research artifacts directly into a submission-ready paper. Applied as a unified pipeline to the APTOS 2021 Big Data Competition on OCT-based diabetic macular edema treatment response prediction, the system surpasses the competition's 1st-place solution (Team BlueSky) on all three tasks: **Task 1 Mean AUC 0.9636 vs 0.9225 (+4.5% relative), Task 2 calibration tolerance 0.606 vs 0.5906 (+2.6%), Task 3 CI AUC 0.8067 vs 0.78 baseline**, across 99+ total experiments spanning four autonomous research sessions. This paper itself was generated by Paper Orchestra from the research artifacts produced during those sessions — the full loop, from raw data to submitted manuscript, required no manual scientific writing.

## Problem Statement

The standard ML research cycle is bottleneck-heavy at every stage:
- **Experiment implementation**: human writes code for each variant
- **Execution**: human submits jobs and monitors
- **Analysis**: human interprets results and decides next steps
- **Writing**: human converts results into a manuscript

Existing LLM coding agents (Claude Code, GitHub Copilot, Cursor) automate individual coding tasks but do not maintain strategic research context across experiments, autonomously decide the next hypothesis, or close the loop from results to publication. We address all three gaps.

## System Architecture

### Component 1: OpenClaude Scientific Harness

**OpenClaude** is an open-source, multi-provider LLM coding agent CLI forked from Claude Code, with proprietary telemetry removed and support for self-hosted LLM backends via OpenAI-compatible APIs. We deploy it against a local vLLM server running **Qwen3.5-27B** on 4× NVIDIA H100 80GB GPUs at `http://localhost:8001/v1`.

The **Scientific Harness** wraps OpenClaude with a two-file structured context system per task:

**CLAUDE.md (Domain Memory)** — contains:
- Research objective and primary metric
- Dataset statistics, preprocessing details, hardware constraints, conda environment
- Accumulated experiment insights written after each experiment (e.g., "HRF pos_weight < 1.0 causes AUC collapse to 0.50"; "MixUp is the only regularization that helps"; "epoch count dominates resolution under time budget")
- This file grows during the session, compounding the agent's domain knowledge

**program.md (Research Constitution)** — contains:
- Hard constraints: forbidden code paths (eval manipulation, checkpoint reuse, new package installs, TIME_BUDGET modification)
- Autonomy mandate: "NEVER STOP — run until manually interrupted; if ideas run dry, reason harder; never ask the human for permission"
- Scope: the agent owns `train.py` completely; all other files are read-only
- Simplicity bias: "if two configs perform equally, the simpler one wins"

The agent reads both files at each turn, implements the next experiment by modifying `train.py`, executes via subprocess (`conda run -n aptos2021 python train.py`), parses output, writes to `results.json` (structured JSON) and `changelog.md` (narrative reasoning), then immediately proposes and implements the next experiment.

### Component 2: AutoResearch Loop

The AutoResearch loop is the operational layer — it requires no orchestration code beyond the harness context. The agent:
1. Reviews `results.json` for all prior experiment metrics and `changelog.md` for accumulated reasoning
2. Forms a hypothesis (written first to `changelog.md` as a chain-of-thought)
3. Implements the hypothesis in `train.py`
4. Runs training (TIME_BUDGET-constrained: 900 seconds per Task 1/2 experiment)
5. Parses structured output into `results.json` and a narrative decision (KEEP/REVERT with rationale) into `changelog.md`
6. Archives code (`archive/train_expN.py`) and logs (`archive/log_expN.txt`) for reproducibility
7. Immediately begins the next iteration

Four research sessions were conducted across four task-specific environments:
- **Session 1 (2026-04-21)**: Project setup, BlueSky architecture reverse-engineering from arXiv:2505.05768
- **Session 2 (2026-04-26)**: Task 1 AutoResearch — 14 experiments (Exp #0–#13), 01:45–10:50, ~9 hours unattended
- **Session 3 (2026-04-29)**: Task 1 continuation (Exp #14–#32, 18 additional), Task 2 AutoResearch (14 versions), Task 3 initialization
- **Session 4 (2026-05-04)**: Task 2 BlueSky surpass, Task 3 full pipeline (53 experiments), OLIVES cross-dataset validation, significance report generation

### Component 3: Paper Orchestra

After the research sessions complete, **Paper Orchestra** — a multi-agent LaTeX generation pipeline running on Claude — ingests two research artifacts: `idea.md` (this file) and `experimental_log.md` (structured experiment records). It executes five sequential/parallel stages via dedicated subagents:
1. **Outliner**: generates section-by-section JSON outline
2. **Lit Reviewer** (parallel): WebSearch → Semantic Scholar verification → BibTeX → Intro/Related Work
3. **Figure Generator** (parallel): PaperBanana CLI with VLM critic for each figure
4. **Section Writer**: Abstract, Methodology, Experiments, Conclusion from experimental_log.md numbers
5. **Reviewer + Refiner** (loop): simulated peer review → revision → re-review until score improves or max_refine reached

This paper is the demonstration artifact of that pipeline — the text, structure, figures, and bibliography were generated without manual scientific writing.

## Case Study: APTOS 2021 OCT Biomarker Challenge

### Task Overview

The APTOS 2021 Big Data Competition requires predicting DME treatment responses from OCT images across three interdependent tasks:

**Task 1 — Multi-label Biomarker Classification (IRF/SRF/PED/HRF)**
- 221 patients, 2,875 OCT images (1264×596 JPG, right half = OCT B-scan)
- Multi-Instance Learning (MIL): 3,301 bags (case-level from Stage 1 + image-level from Stage 2)
- Extreme class imbalance: IRF 84.1%, SRF 34.2%, PED 18.3%, HRF 95.7%
- Metric: Mean AUC across all 4 classes

**Task 2 — CST Regression (Central Subfield Thickness)**
- Predict absolute CST (μm) from OCT center image
- Custom metric: calibration tolerance (fraction within ±7.5% relative error)
- BlueSky used ResNet101 + custom MAPE+SmoothL1 loss + progressive resolution

**Task 3 — VA Regression & CI Classification**
- Visual Acuity (VA) regression and Continue Injection (CI) binary classification
- Features: Task 1 biomarker predictions + Task 2 CST prediction + 15 sponsor metadata features
- Classical ML: VotingRegressor/Classifier ensemble

### Key Findings Discovered by Science Agent

**Finding 1 — Depth Beats Capacity Under Time Budget**
Under a 900-second TIME_BUDGET, ConvNeXt-Tiny (28M params, 90s/epoch) outperforms Swin-Base (87M params, 150s/epoch): 17 epochs vs 2 epochs enables far more gradient updates. This non-obvious insight — that compute efficiency of architecture matters more than raw capacity when wall time is fixed — was discovered by the agent after Exp #3 (Swin-Base + Attention MIL) and confirmed across subsequent experiments.

**Finding 2 — MixUp is the Only Effective Regularization**
Of 7 regularization techniques tested (dropout increase, label smoothing, SWA, TTA, weight decay, focal loss, pos_weight), MixUp (α=0.2, prob=0.5) is the sole technique that improves Mean AUC (+0.0057). All others degrade performance, and several catastrophically (label smoothing: −0.0149; SWA at HRF: 0.7953). The regularization budget is narrow: MixUp + dropout 0.3 is the ceiling.

**Finding 3 — HRF pos_weight < 1.0 Causes Learning Collapse**
Downweighting the 95.7%-prevalence HRF class below 1.0 causes complete HRF AUC collapse to 0.5025 (Exp #6). The agent identified this as a critical negative result after observing the pattern at epoch 10 vs epoch 5, and propagated the constraint into CLAUDE.md for all subsequent experiments.

**Finding 4 — EMA Stabilizes Imbalanced Class Dynamics**
Exponential Moving Average (decay=0.999) of model weights dramatically stabilizes the volatile HRF AUC (+0.0086, from 0.83–0.88 range to stable 0.91–0.92), discovered in autoresearch_task1 Exp #22. EMA acts as a temporal ensemble, smoothing the sharp peak-then-collapse training dynamics universal to all 32 experiments.

**Finding 5 — Per-Class Epoch Selection is a Free Lunch**
Different biomarkers converge at different epochs: HRF peaks early (Epoch 2–3), PED peaks later (Epoch 4–6). Selecting the per-class best checkpoint before ensembling yields +0.0036 AUC at no additional compute cost.

**Finding 6 — Calibration Dominates Architecture for Regression**
Post-hoc calibration (3-segment piecewise linear, pw3) improves Task 2 cal_tol from raw 0.091 (384px) to 0.606 — a 566% improvement. Architecture changes across 14 versions contributed less than calibration method selection. However, this is conditional: on the OLIVES dataset where Pearson r > 0.97, the same calibration hurts (0.8298 → 0.5461), indicating pw3 overfits on small calibration sets (n=164).

**Finding 7 — Upstream Noise Asymmetry in Multi-Task Pipelines**
VA regression is robust to CST prediction noise (MAE actually improves from 0.1466 to 0.1435 with predicted vs ground-truth features), while CI classification is fragile (AUC drops 0.7799 → 0.6801). Root cause: VA is anchored by the stable preVA clinical feature (34.5% importance), while CI depends critically on calibrated CST accuracy.

## Significance

This work demonstrates that the combination of (a) a structured two-file harness context, (b) an open-weight LLM agent with full code ownership and an autonomy mandate, and (c) an automated manuscript pipeline constitutes a viable research acceleration system that:
- Runs 99+ experiments across 3 tasks in 4 sessions without human implementation effort
- Discovers non-obvious research insights (depth > capacity, HRF collapse, calibration dominance)
- Surpasses a competition-winning solution on all tasks
- Produces the submission manuscript automatically from research artifacts

The harness design is domain-agnostic and deployable on any institution's hardware using open-weight models. All code, experiment archives (train_expN.py for all 99+ experiments), structured result logs (results.json), and changelogs (changelog.md) are released for reproducibility.
